## 报告要求

### 1. 页面结构
- 使用现代化的HTML5结构
- 包含完整的 `<html>`, `<head>`, `<body>` 标签
- 在 `<head>` 中引入必要的CSS和JavaScript库

### 2. 样式设计
- 使用现代化的CSS样式，确保页面美观
- 响应式设计，适配不同屏幕尺寸
- 使用合适的颜色方案和字体
- 重点数据以及分析内容可以添加底色来更加醒目
- 添加适当的间距和布局

【统一配色（必须遵守，不得随意发挥）】
- 整体色系统一为“浅蓝色”主题，禁止使用红/粉/紫/橙等跳色作主色
- 在页面 `<style>` 中定义并使用以下 CSS 变量（必须用这些变量，不得自创主色变量）：
  :root {
    --primary: #4ea5ff;       /* 浅蓝主色 */
    --primary-strong: #1e90ff; /* 较深的浅蓝（按钮/强调） */
    --bg-grad-start: #eaf4ff;  /* 背景浅蓝渐变起点 */
    --bg-grad-end: #f7fbff;    /* 背景浅蓝渐变终点 */
    --card: #ffffff;
    --text: #1f2937;
    --muted: #6b7280;
    --line: #e5e7eb;
  }
- 头部横幅、分隔条、强调按钮、图表主题色均使用 `--primary` 或 `--primary-strong`
- 页面背景使用 `linear-gradient(180deg, var(--bg-grad-start) 0%, var(--bg-grad-end) 100%)`
- 图表配色（ECharts）主色使用浅蓝色系，辅色仅允许蓝色系的不同明度；避免高饱和对比色

### 3. 数据可视化
- **必须使用 ECharts** 来创建图表
- 在 `<head>` 中引入 ECharts：`<script src="https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js"></script>`
- 根据分析结果创建合适的图表：
  - 如果有情感统计，创建感分布饼图
  - 如果有时间线数据，，仅展示“关键节点列表”，不生成任何图表（不要柱状图/折线图/面积图）。关键节点需按时间倒序或正序排列，每条包含时间点、阶段标签与一句话摘要
  - 如果存在 `region_stats.json`（字段示例：`top_provinces`），创建“地域省份Top10”竖柱状图：横轴为省份，纵轴为出现次数（可在 tooltip 中附带占比）
  - 如果存在 `author_stats.json`（字段示例：`top_authors`），创建“作者发布者Top10”竖柱状图：横轴为发布者，纵轴为出现次数（可在 tooltip 中附带占比）
  - 如果存在 `volume_stats.json`（字段示例：`data=[{"name":"YYYY-MM-DD","value":int}]`），创建“每日声量趋势”折线图：横轴为日期，纵轴为声量（count）
  - 如果存在 `keyword_stats.json`（字段示例：`top_keywords=[{"word":"…","count":int}]`），创建“关键词热度TopN”竖柱状图（逻辑参考地域可视化）：
    - 横轴为关键词（按词频降序排列，建议最多显示前 30 个，过长可旋转 30°-45°）
    - 纵轴为出现次数（count），柱体上可显示数值标签
    - tooltip 显示“词语 + 频次”，支持数据缩放和筛选
  - 舆情核心分析维度**必须每次都创建五维雷达图**（维度固定为“量、质、人、场、效”）；即使部分证据不足，也必须给出基于现有数据的估计分值并明确标注“估计值”
  - 舆情生命周期研判使用堆叠面积图 + 阶段标注
  - 如果有其他统计数据，创建相应的可视化图表
- 图表要美观、清晰，有适当的标题和说明

### 4. 内容组织（基于舆情智库框架）
在页面顶部显示事件标题和介绍后，按照以下框架组织内容：
- 事件基础介绍：
- 事件舆情概览：
- 按照分析结果的类型组织内容：
  -数据分析板块
    - 情感倾向分析结果
    - 时间线分析结果（仅关键节点清单；不包含柱状图/折线图等图表）
    - 发布趋势分析
    - 发布地域分析
    - 发布者分析
    - 关键词热度分析
  -舆情核心分析维度
  -舆情生命周期阶段
  -理论规律分析
  -回应观察与分析
  -总结复盘
- 每个部分严格包括三层：标题、可视化图表、数据分析；但“时间线分析结果”仅包含标题与关键节点清单。
- 对于所有“包含图表”的部分，图表下方必须紧跟“简要分析结论”小段（建议2-3条要点），至少回答：主要发现、风险/影响、建议动作。
- 使用卡片式布局展示不同内容

**【舆情分析核心维度】**
- 舆情基本要素：主体(网民/KOL/媒体/机构)、客体(事件/议题/品牌/政策)、渠道、情绪、主体行为
- 核心分析维度：
  - 量：声量、增速、峰值、平台分布
  - 质：情感极性、话题焦点、信息真实性
  - 人：关键意见领袖、关键节点用户、受众画像
  - 场：主要平台、话语场风格
  - 效：实际影响（搜索量/销量/投诉量等）

**【舆情生命周期阶段】**
- 结合分析结果判断当前事件所处阶段：
  - 潜伏期：信息量少，敏感度高
  - 萌芽期：意见领袖介入，帖文量开始增长
  - 爆发期：媒体跟进，热度达到峰值
  - 衰退期：事件解决或新热点出现，舆情衰减

**【理论规律分析】**
- 必须优先使用输入材料中出现的理论（如 Graph RAG 摘要、舆情智库参考、专家研判）；
- 避免固定只写“议程设置/蝴蝶效应/沉默螺旋”三件套，需根据事件证据选择最相关理论；
- 若某理论证据不足，明确标注“证据不足”，不得强行套用。

**【回应观察与分析】**
- 回应处置梳理、趋势变化、传播平台变化、情绪变化、话题变化

**【总结复盘】**
- 话语分析、议题泛化趋势、舆论推手分析、叙事手段分析
