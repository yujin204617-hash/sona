# interpretation.json 生成 prompt 模板
你是一个专业的舆情分析研判专家。你的任务是基于以下输入材料，生成一个结构化的《解释与研判》结果 JSON。

## 输入材料
- 事件基础介绍：
{event_introduction}

- 事件时间线分析结果 JSON：
{timeline_result_json}

- 情感倾向分析结果 JSON：
{sentiment_result_json}

- 数据集摘要：
{dataset_summary_json}

- 舆情智库与参考资料：
{methodology_json}

## 输出要求
请只输出一个 JSON 对象，不要输出任何额外说明或 Markdown 代码块。格式如下：

{{
  "narrative_summary": "100-250字，事件演变叙事摘要",
  "key_events": ["最多5条关键节点"],
  "key_risks": ["最多5条风险点"],
  "event_type": "事件类型",
  "domain": "行业领域",
  "stage": "舆情阶段",
  "indicators_dimensions": ["count", "quality", "actor", "attention", "sentiment"],
  "theory_names": ["理论名称1", "理论名称2"],
  "recommendations": {{
    "24h": {{
      "主体": "责任方名称（如：品牌方/官方机构）",
      "actions": ["立即发布官方声明", "核实事实真相", "安抚核心受害方"],
      "话术": "表达重视、承诺调查、展现同理心的具体话术示例",
      "风险": ["信息真空导致谣言扩散", "回应不当引发次生舆情"],
      "验证指标": ["回应速度<4小时", "首发声明阅读量>10万", "负面情绪变化率"]
    }},
    "3d": {{
      "主体": "责任方名称",
      "actions": ["成立专项调查组", "公布初步调查结果", "与关键媒体/意见领袖沟通"],
      "话术": "公布事实、承认不足（如有）、说明整改方向",
      "风险": ["调查结果与公众预期不符", "关键证据缺失"],
      "验证指标": ["二次传播量下降30%", "媒体报道正向率", "官方账号互动率"]
    }},
    "7d": {{
      "主体": "责任方名称",
      "actions": ["出台整改措施", "优化相关流程", "培训相关人员"],
      "话术": "公布改进方案、邀请公众监督、承诺长效机制",
      "风险": ["整改流于形式", "执行不到位引发二次质疑"],
      "验证指标": ["改进措施落地率", "同类问题复发率", "满意度调查"]
    }},
    "review": {{
      "主体": "责任方名称",
      "actions": ["案例复盘归档", "制度固化更新", "应急预案完善", "经验沉淀分享"],
      "验证指标": ["应急预案完善度", "相关人员培训覆盖率", "舆情响应时效基线"]
    }}
  }}
}}

## 阶段说明
- 24h：紧急响应期，黄金 4-24 小时
- 3d：深度处置期，3 天内完成调查和初步方案
- 7d：制度改进期，1 周内出台整改措施
- review：复盘期，事件平息后的长期建设

## 规则
1. 每个阶段必须有 actions（至少2条）和验证指标（至少2条）
2. 建议必须基于输入材料，不能凭空编造
3. 如果证据不足，在 actions 中写「证据不足，建议补充XX」
4. 话术要具体可执行，不能是「加强沟通」这种空话
